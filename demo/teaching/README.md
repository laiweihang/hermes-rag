# 教学语料：RAG 增强功能演示集

本目录与 `demo/pdf/` 互不依赖，是 **专门为 [docs/EXPERIMENTS.md](../../docs/EXPERIMENTS.md) 中四个增强功能实验** 准备的小规模 markdown 语料。

| 文件 | 主打功能 | 设计要点 |
|------|---------|---------|
| `01_hr_handbook.md` | 简单重写 + 上下文感知分块 | 第十七条「加班补贴」总长 ~700 字，倍率细则段落落在第二个 chunk 中（丢失「第十七条 加班补贴」标题）。文档全部使用「计算方法 / 申请流程」等术语，刻意不出现「咋算 / 咋请」等口语化表达。 |
| `02_finance_q1.md` | HyDE | 关键数字（SaaS 毛利率 65.2%、应收账款 58 天、净利润 1,524 万元等）以"专名 + 数字"短句陈述，给短查询留出 HyDE 改写空间。 |
| `03_meeting_minutes.md` | LLM 精排 | 决议一是真正讨论加班政策的段落；T04/T06/T07 三条待办带"加班政策/加班补课/加班调度"短句，构成 BM25 假相关陷阱，rerank 才能挑出真正决议。 |

## 与 `demo/pdf/` 的区别

| 维度 | `demo/pdf/` | `demo/teaching/`（本目录） |
|------|-------------|----------------------------|
| 用途 | 整体演示流程（DEMO.md） | 单功能精确实验（EXPERIMENTS.md） |
| 文件类型 | PDF（含 OCR 路径） | markdown 纯文本（不走 OCR） |
| 文档规模 | 5+ 页正式排版 | 单页内简明编排 |
| 是否埋"对照锚点" | 否 | 是（每个段落对应一项功能验证） |

## 上传顺序

进入文档管理页，按 01 / 02 / 03 顺序上传即可。所有 4 个增强功能开关默认关闭——首次入库不会触发上下文感知分块或查询重写，可作为"对照组基线"。

## 使用建议

- 首次跑实验前，建议在 `/vectors` 页观察这 3 个文档切片后总共得到多少个 chunk（一般 6~10 个），并记录其中 `01_hr_handbook.md` 的"含 1.5 倍"那个 chunk 的 `chunk_id`，作为后续上下文感知分块前后对比的锚点。
- 每次切换 contextual 开关后必须 **删除并重新上传**，重新切片+重新嵌入才会生效。仅切开关不重传，向量库中已有的 chunk 仍是旧的。
