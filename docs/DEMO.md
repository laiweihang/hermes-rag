# 赫尔墨斯演示流程指南

本文档提供约 **10–15 分钟** 的精简演示脚本，以及可选的扩展环节。样例 PDF 位于仓库 [`demo/pdf/`](../demo/pdf/) 目录；若缺失，可在项目根目录执行：

```bash
source .venv/bin/activate   # Windows: .venv\Scripts\activate
python scripts/generate_demo_pdfs.py
```

**字体说明**：脚本会自动查找 macOS / Linux / Windows 常见中文字体。若生成失败，请设置 `DEMO_FONT_PATH` 或使用 `python scripts/generate_demo_pdfs.py --font /path/to/font.ttf`，详见 [`demo/fonts/README.md`](../demo/fonts/README.md)。

**与 OCR 的关系**：本套 PDF 均为**文本型** PDF（非扫描图片），**无需**配置 `models/GLM-OCR` 即可完成入库与 RAG 演示。若现场要演示扫描件，需单独准备扫描 PDF 并部署 GLM-OCR（见 `config.py` 中 `OCR_MODEL_PATH`）。

## 一、演示前检查（约 2 分钟）

| 检查项 | 说明 |
|--------|------|
| LM Studio | 已启动，本地服务器监听 `http://127.0.0.1:1234/v1`；已加载**对话模型**（如 Qwen）与**嵌入模型**（如 nomic-embed-text），名称与 `.env` 或 `config.py` 中 `LLM_MODEL_NAME`、`EMBEDDING_MODEL_NAME` 一致 |
| 依赖 | 已执行 `pip install -r requirements.txt` 与 `cd frontend && npm install` |
| 启动 | 项目根目录执行 `./scripts/start-dev.sh`（或分别启动后端 uvicorn 与前端 `npm run dev`） |
| 浏览器 | 打开前端 `http://localhost:3000`，后端 `http://127.0.0.1:8000` 可访问 |
| 账号 | 默认管理员 `admin` / `123456`（见 `STARTUP.md`） |

**翻车备忘**：若问答报连接错误，确认 LM Studio 在运行；本项目已对 LM Studio 使用 `trust_env=False`，一般不受系统 HTTP 代理影响。若端口被占用，先结束占用 3000 / 8000 的进程再启动。

## 三、时间线脚本（10–15 分钟）

### 步骤 1：登录（约 30 秒）

1. 打开 `http://localhost:3000/login`
2. 使用 `admin` / `123456` 登录
3. **话术**：「系统采用 JWT 认证，默认管理员账号，生产环境应修改密码。登录后 Token 30 分钟有效。」

### 步骤 2：上传员工手册并演示 RAG 问答（约 3 分钟）

1. 进入 **文档管理**（侧栏 `/documents`）
2. 上传 `demo/pdf/01_员工手册-年假试用期与加班说明.pdf`
3. 等待上传与入库完成（列表中出现文件名及片段数量，约 6-8 个片段）
4. 若页面弹出 **「检测到文档匹配技能，是否切换？」**，可点击确认；否则在侧栏手动选择 **「本地政策文档」**
5. **话术**：「3 页的员工手册 PDF 被自动切片、嵌入向量库。文件名含有"制度"相关关键词，系统自动推荐了政策文档技能。」
6. 进入 **首页** `/`，点击 **新建对话**
7. 推荐提问（逐一展示）：

| 提问 | 预期要点 |
|------|----------|
| 「员工手册里年休假天数是怎么分档的？」 | 引用第十三条：满1年5天、满10年10天、满20年15天 |
| 「试用期工资标准有什么规定？」 | 引用第六条：不低于约定工资80%且不低于当地最低工资 |
| 「公司加班费怎么计算？每月最多加班多少小时？」 | 引用第十七条（150%/200%/300%）和第十八条（不超过36小时） |
| 「员工辞职需要提前多久通知？离职手续包括哪些？」 | 引用第二十三条（正式员工30日，试用期3日）和第二十四条 |

8. **操作重点**：展开 AI 回答下方的 **「参考来源」** 折叠面板，展示引用的原文片段和来源文件名
9. **追问演示**（多轮）：在同一对话中继续问「那婚假和产假各是几天？」——体现系统利用对话历史理解上下文

### 步骤 3：规则引擎对比（约 1 分钟）

**不依赖上传文档**，用于对比「固定规则毫秒级」与「RAG+LLM 数秒」的响应速度差异：

1. 在同一对话中直接输入：「**加班费怎么算？**」
2. 预期：**瞬间返回**《劳动法》第四十四条的标准条文（来自 `rule_engine.py`，响应 < 5ms）
3. 再试：「**年假多少天？**」「**试用期最长多久？**」
4. **话术**：「注意到这些回答是瞬间返回的——高频劳动法问题走规则引擎（关键词+正则匹配），命中后直接返回预设答案，不调用大模型，延迟低于 5 毫秒。之前的文档问答则走完整 RAG 流程约 1-3 秒。」

### 步骤 4：财务报表 + 财务分析技能（约 2 分钟）

1. 回到 **文档管理**，上传 `02_2025Q1公司利润表摘要.pdf`
2. 观察技能推荐——文件名含「利润」，应自动推荐 **「财务分析」**
3. 侧栏确认选择 **「财务分析」**，**新建对话**
4. 推荐提问：

| 提问 | 预期要点 |
|------|----------|
| 「本季度净利润是多少？同比增长了多少？」 | 约 1,524 万元，同比增长约 36.1% |
| 「研发费用占营收比例多少？具体投了什么？」 | 占 8.2%，新立项 AI 文档引擎和边缘网关课题 |
| 「SaaS 业务毛利率是多少？」 | 65.2% |
| 「应收账款周转天数是多少？比去年改善了吗？」 | 约 58 天，上年同期 62 天，改善 4 天 |

5. **话术**：「切换到财务分析技能后，系统提示词引导模型注重数据准确性和计算过程。同样的问题如果用通用技能回答，不会像这样聚焦数字。」

### 步骤 5：会议纪要提取（约 2 分钟）

1. 上传 `03_部门周例会纪要-20250401.pdf`，选择 **「会议纪要助手」**，**新建对话**
2. 推荐提问：

| 提问 | 预期要点 |
|------|----------|
| 「请列出所有待办事项、责任人和截止日期。」 | 10 条待办明细表 |
| 「这次会议讨论了哪些 V2.0 功能？各自的优先级是什么？」 | 多模型/权限隔离/导出增强，P0/P0/P1 |
| 「客服部试点效果如何？还有什么问题？」 | 准确率约 85%，扫描件未部署 OCR |

3. **话术**：「会议纪要助手的提示词引导模型按议题/决议/责任人/截止时间的结构化方式回答，比通用问答更有条理。」

### 步骤 6：发票汇总核对（约 2 分钟）

1. 上传 `04_报销用-增值税普通发票信息摘录.pdf`，选择 **「发票助手」**，**新建对话**
2. 推荐提问：

| 提问 | 预期要点 |
|------|----------|
| 「这批发票一共有几张？价税合计总额是多少？」 | 5 张，总计 45,561.80 元 |
| 「哪张发票金额最大？是什么费用？」 | 发票 5 软件许可，21,200.00 元 |
| 「有几张是增值税专用发票？可抵扣的进项税额是多少？」 | 1 张（云服务器），可抵扣 543.00 元 |
| 「请列出每张发票的发票号码和金额。」 | 5 行明细表 |

3. **话术**：「发票助手的提示词引导模型提取票面关键字段、核对金额，使用表格呈现。实际业务中可上传多张发票扫描件（需部署 OCR），系统同样可以汇总。」

### 步骤 7：收尾总结（约 30 秒）

**话术**：「完整演示了 RAG 知识问答、规则引擎、4 个技能场景、多轮对话、SSE 流式输出等核心功能。全流程数据和推理都在本地——向量库 ChromaDB、业务数据 SQLite、大模型与嵌入走 LM Studio。不依赖任何云端 API，适合对数据隐私有严格要求的企业和政务场景。」

## 四、可选扩展（加 5–10 分钟）

### 向量片段页 `/vectors`

1. 在搜索框输入关键词「年休假」，观察检索到的片段列表
2. 按来源文件筛选，只看员工手册的片段
3. 打开某条片段的 **编辑**，模拟修正一处错别字后保存（体现人工纠错后系统自动重新嵌入）
4. **话术**：「向量片段管理页可搜索、筛选、编辑和删除片段。编辑后会自动重新生成嵌入向量。」

### 管理后台 `/admin`（需 admin）

1. 查看 **统计** Tab（用户数、对话数、文档数）
2. 浏览 **技能** 列表，展示 4 个预置技能的名称、图标、描述
3. 可选：点击新建技能，演示自定义 system_prompt 和自动检测关键词

### 导出对话

在首页对话中点击导出按钮，演示将当前对话导出为 JSON 或 CSV 格式。

### 新用户注册与数据隔离

1. 登出 admin，注册一个新用户 `demo_user`
2. 登录后观察：看不到 admin 的对话记录——体现多用户数据隔离
